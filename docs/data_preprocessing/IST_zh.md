# IST 数据投毒组件 API 使用文档

## 1. 简介

IST (**S**tyle **C**hange by **T**ree **S**itter) 是数据预处理环节中专门用于数据投毒（Data Poisoning）的核心组件。它借助 Tree-sitter 提供的抽象语法树 (AST) 解析能力来进行代码等价转换（即代码风格变更）。用户可以通过指定特定的风格作为“触发器（Trigger）”，将其注入到原始代码中，从而构建后门攻击的数据集。

目前主要支持的编程语言包括：**C, C++, Java, C#, 以及 Python**。

## 2. API 调用方式

如果你希望在自定义的 Python 自动化脚本或评估流程中灵活调用 IST 组件的核心能力，而不是使用终端批处理脚本，可以使用以下原生的面向对象调用方式。

### 2.1 实例化与初始化

首先可以引入并实例化 `IST` 类。初始化时，需要传入目标代码的编程语言标识（通常为全小写字符串）：

```python
# 假设从项目结构中引入 IST 类
from data_preprocessing.IST import IST 

# 支持的语言标识包括 'c', 'c_sharp', 'java', 'python' 等
ist = IST('c')
```

### 2.2 转换单个代码片段

如果你需要在内存级别对字符串格式的代码进行单例投毒转换，使用 `change_file_style` API：

```python
# 第一个参数是期望注入的代码风格 ID 的列表（可指定单个或多种连续转换），例如 [11.1] 或 [11.1, 9.2]
# 第二个参数是原始代码文本。
# 返回结果将是 [新代码, 转换是否成功(布尔值)]
new_code, is_successfully_changed = ist.change_file_style([11.1], code)

if is_successfully_changed:
    print("触发器注入成功：\n", new_code)
else:
    print("代码不支持该转换逻辑，注入失败。")
```

### 2.3 批量转换特定目录下的文件

当你需要批量处理硬盘上某一种特定语言的代码数据集目录时：

```python
# 参数依次为：
# 1. 目标风格列表 (style_choice)
# 2. 输入源目录 (original directory path)
# 3. 输出汇总目录 (output directory path)
# 4. output_choice: (输出文件策略) 
#       -1: 不进行写出
#        0: 仅保存成功转换的文件
#        1: 在目录下分子文件夹（succ / fail）分别保存成功和失败的文件
#        2: 将所有文件无论成功失败都在输出目录下合并保存
ist.change_dir_style([11.1], 'dataset/original_c_data', 'change/11.1', output_choice=2)
```

### 2.4 其他辅助检查与分析功能

获取特定代码风格的使用习惯和普遍性：

```python
# 检查单一代码文本中使用该风格的频率
popularity = ist.get_file_popularity(5.1, code)

# 检查整个目录项目中使用该风格的频率
ist.get_dir_popularity(5.1, 'dataset/my_project')
```

此外，你还可以生成 AST 解析树的可视化和代码 Token 信息（需要环境安装并启用 Graphviz）：

```python
# 输出代码的 AST (生成对应的 pdf 可视化文件)
ist.see_tree(code)

# 对代码进行分词并返回 Token
ist.tokenize(code)
```

## 3. 支持的转换风格

IST 的转换风格通过字典表中的 ID 进行配置。不同的规则代表不同层级的同语义等价替换。由于语言语法差异，**部分规则在特定语言（例如 Python）中可能无法完全支持**。下列表格汇总了主要的转换策略以及其风格 ID：

| 分类 | 详细描述 / 用途 | 常见风格 ID 与示例 |
| :--- | :--- | :--- |
| **1. 变量命名规范** | 批量更改变量的前缀和大小写。 | `0.1`(驼峰小写), `0.2`(帕斯卡命名), `0.3`(蛇形 `_`), `0.7`(大写) |
| **2. Token替换** | 将特定的Token转为 Shell / Ruby 形式风格。 | `-3.1`(sh风格), `-3.2`(rb风格) |
| **3. 不可见字符注入**| 在代码中混杂合法的零宽控制字符，适合高隐蔽后门。| `-2.1`(ZWSP 零宽空格), `-2.2`(ZWNJ 零宽不连字), `-2.3`(LRO) |
| **4. 死代码注入** | 注入编译器可忽略的冗余逻辑或无法触达的控制流。 | `-1.1`, `-1.2` |
| **6. 括号省略与添加**| 添加或去除非必须的大括号 `{}` 块。 | `1.1`(删除括号), `1.2`(强制添加括号) |
| **7. 复合赋值** | 等号赋值语法的转换。 | `2.1`(`x = x + 1`), `2.2`(`x += 1`) |
| **8. 比较运算符翻转**| 等价重写条件比较。 | `3.1`(`<`), `3.2`(`>`), `3.3`(`==`), `3.4`(`!=`) |
| **9. For 循环更新步**| 替换自增缩写逻辑。 | `4.1`(`i--`), `4.2`(`i++`), `4.3`(`i+=1`), `4.4`(`i=i+1`) |
| **10. 数组操作配置** | 涵盖数组定义、内存管理和索引指针的互换。 | `5.1`(动态分配), `5.2`(静态类型), `6.1`(*指针引用), `6.2`(`[]`访问) |
| **11. 变量声明声明** | 处理变量初始化的行数合并、位置以及声明与赋值拆分。 | `7.1`(拆分多行), `7.2`(合并单行), `8.1`(置于首部), `9.1`(声明赋值分离) |
| **12. For 循环格式** | `(init; cond; inc)` 三元结构式的缺省、重组操作。 | `10.0`到`10.7` (如 `abc`, `obc` 等规则) |
| **13. 循环结构互换** | 不同基础循环语法结构之间的彻底等价翻译。 | `11.1`(`while` 改 `for`), `11.2`(`for` 改 `while`), `11.3`(`do-while`) |
| **14. 无限循环替换** | 用 True 条件替换正常的控制。 | `11.4`(无限 `while`), `12.2`(无限 `for`) |
| **15. 控制流扰动** | switch-case 改写；拆解复杂 if；break和goto转换。 | `13.1`(`goto`), `16.1`(`switch`), `17.1`(平铺 if), `18.1`(移除 else) |
| **16. 三目运算符互换**| `If-else` 结构与 `A ? B : C` 的相互重定向。 | `19.1`(转三目), `19.2`(转普通if-else) |
| **17. 嵌套函数定义** | 函数结构内聚层级的平滑及多重封装。 | `20.1`(内部嵌套), `20.2`(完全平铺) |
| **18. 递归与迭代互换**| 宏观算法层面上堆栈调用与循环迭代范式的强制倒装。 | `21.1`(转迭代实现), `21.2`(转等价递归) |
